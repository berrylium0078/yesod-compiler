# 多项式类型扩展

## 多项式类型说明

多项式底层存储为堆分配的动态 `mint` 数组：`{ mint *a; int n; }`，表示 `a[0]*x^0 + a[1]*x^1 + ... + a[n-1]*x^(n-1)`，下面我们称 `n` 为该多项式的长度。

其中 `mint` 表示模 `998244353` 质数域的类型，存储为 `int32`，支持高效的 `+ - *` 运算。

## 语义规范细化

本节补全文档中未显式规定的语义规则，所有实现和测试必须遵守。

### 类型定位

- `poly` 是一等（first-class）非标量类型。不能自动用于标量语境（如 `if(p)`、`while(p)`、`&&` / `||` 的操作数）。
- `!p` 不支持 `p: poly`。多项式算法需要显式维护目标项数或边界。
- `p[k]` 的结果类型为 `mint`，`p[n,m]` 的结果类型为 `poly`。
- `poly(int)` 和 `poly(mint)` 均为显式构造，不接受隐式 `int → poly` 转换（除非将来另行扩展）。

### 对象身份与别名规则

每个 `poly` 值在运行时对应一个堆分配对象（backing allocation）。该对象的身份在 SSA 分析中可被追踪。

**被调用者所有（`e` 模式）的参数在入口处必须绑定到独占对象。** 编译器必须从调用点保证：
1. 同一个 `e` 实参不能被同一调用的多个 `e` 形参共享。
2. 传递给 `e` 的值不能与调用后仍被读取的任何其他 `poly` 共享同一底层对象。
3. 如果编译器无法静态证明 <1> 和 <2>，则**必须**在调用前插入深层复制（deep copy）。

**调用者所有（`r` 模式）的参数允许别名。** `r` 模式是借用，被调用者不得释放或修改其内存。

#### 存储空间独立规则（v2）

在函数内部视角，不同来源的 poly 值具有不同的存储空间独立性保证：

| 来源 | 存储独立性 | 说明 |
|------|-----------|------|
| 局部变量 / 表达式临时值 | 独立 | 每次分配均有独立堆内存 |
| `e` 模式参数 | 独立 | 调用方保证已复制或转移独占对象 |
| `r` 模式参数 | 可共享（别名） | 多个 `r` 参数可指向同一底层对象 |
| 全局变量 / 全局数组 | 独立 | 与局部/参数均不共享 |
| 函数数组参数（形参为数组） | 独立 | 与局部/参数均不共享 |

**别名边界规则：** `r` 模式参数之间可以共享存储，但 `r` 模式参数不得与以下任何来源共享存储：`e` 参数、全局变量、全局数组、函数数组参数。若调用方违背此规则，则必须在调用前插入复制以确保 callee 视角的合规性。

### 首版排除项

以下内容在首版中不做支持，除非将来另行指明：
- 全局 `poly` 变量或常量。
- 元素类型为 `poly` 的全局/静态数组。
- 隐式 `int → poly` 类型转换（`poly(exp)` 显式构造是唯一途径）。

> 注意：**局部**元素类型为 `poly` 的数组是支持的，离开作用域时需遍历每个元素释放其堆内存（详见"局部 poly 数组的析构"）。

### 内存管理与调用约定

先把代码转化为 SSA 形式。

一个多项式类型的值只会在其变得不再活跃时释放内存。具体来说，有如下几类情况：
1. 该值在当前语句为最后一次使用 - 代码生成中，在使用这个值之后立刻释放其内存。
2. 该值在基本块 B1 结束处跳转到基本块 B2，但是 B2 的基本块参数没有传入该值 - 在 B1 B2 之间插入基本块 B2' 用来释放所有这样的值。

#### 赋值与移动语义（v2）

所有 `poly` 类型的函数返回值均采用**移动语义**——被调用者返回一个 poly 值时，调用方获得该值的所有权。具体规则：

| 返回的表达式类型 | 行为 |
|----------------|------|
| 局部变量 / 表达式临时值 | 复制一次后返回 |
| `e` 模式参数 | 复制一次后返回 |
| 局部数组 `arr` 中取出的一项 `arr[i]` | 将 `arr[i]` 的多项式数据**移动**出来（数组该位置变为零多项式），然后释放数组中每一个剩余 poly 元素的内存（从后往前遍历释放），再将移动出的值作为返回值 |
| `r` 模式参数 | 必须复制（因 `r` 为借用，仅读不释放） |

实现提示：对于局部数组的移动返回，在 `arr[i]` 位置用零多项式（`{nullptr, 0}`）覆盖原值，再执行标准的数组析构（从后往前 free 每个元素），最后将移动出的 `{ptr, len}` 对作为返回值。

#### 传参所有权特化规则（v2）

`poly` 类型的函数参数传递，所有权有如下两种模式。

**1. 调用者所有（`r`）：**
   - 被调用者不能修改，不过由于已经采用了 SSA 只需要确保不会释放这些参数的内存。在活跃变量分析中认定 `r` 参数在可达 `end` 的基本块出口处活跃（详见下方"活跃分析简化"）。
   - 调用者传递参数时，只允许传递局部定义的变量、中间结果和函数参数（我们认为调用者的调用者所有也是调用者所有）。访问数组或全局变量的处理见下方规则表。

**2. 被调用者所有（`e`）：**
   - 被调用者可以任意修改，处理和局部定义的变量一样，由被调用者负责释放内存。
   - 调用者传递参数时，如果值仍活跃（包括全局和数组）则需要先复制再传递，否则可直接将所有权传递出去。
   - 调用者必须保证此参数在入口处绑定到独占对象；若无法静态证明，需先深层复制。

**传参规则表（细化版）：**

| 实参来源 | 特化模式选择 | 复制策略 |
|----------|-------------|----------|
| 全局变量 / 全局数组 / 函数数组参数 | 必须 `e` | **复制一次**（因全局/数组值在调用后仍可能被其他路径访问） |
| `poly` 类型函数参数 / 局部变量 / 中间结果 | 根据活跃分析选 `e` 或 `r` | 若不活跃且该值**只被传参一次**（反例：`f(p+q, p+q)` 中 `p+q` 被传递给两个参数），选 `e` 且不复制；否则选 `r` 且不复制 |
| 局部数组 `arr` 的左值访问（仅下标提取 `arr[i]` 且结果为 `poly`，未参与其他运算） | 全部 `r` 或全部 `e + 复制` | 如果本次调用中**所有**以 `arr` 为基的访问在使用 `poly` 类型的位置，则全部使用 `r` 模式（零复制）；否则全部先复制再使用 `e` 模式 |
| 同一值传递给同一调用的多个参数 | 按最严格约束 | 若多个参数均为 `poly`，根据上述行逐参数判断。若任一参数需 `e` 模式且该值被多于一个参数引用，则需对该参数执行复制以满足互异约束 |

**别名禁止（核心约束）：** 不允许两个 `e` 参数（同一调用）或一个 `e` 参数与一个调用后仍活跃的 `r`/`e` 值指向同一底层分配对象。违反此约束的程序行为未定义。编译器应保守复制。

#### 活跃分析简化（v2）

为降低实现复杂度，采用以下简化方案替代完整的逆向数据流活跃分析：

1. **全 `e` 版本生成**：对每个函数，预先生成一个所有 poly 参数均为 `e` 模式的版本（mask = 全 1）。该版本用于保守推导参数在函数出口处的活跃性。
2. **可达 end 分析**：在 CFG 上标记哪些基本块存在可达函数出口（`end` / `return`）的路径。
3. **简化活跃判定**：
   - 对于可达 `end` 的基本块：其块参数列表中所有 `r` 模式 poly 参数**均视为在出口处活跃**（即不能释放）。
   - 对于不可达 `end` 的基本块（如无限循环、unreachable）：其 `r` 模式 poly 参数视为不活跃，可释放。
   - 对于 `e` 模式 poly 参数：在函数内与局部变量相同处理，**不**在出口处标记为活跃（因被调用者拥有所有权，需负责释放或返回）。
4. **无需迭代求解**：此规则为一次性标记，不依赖数据流迭代，因此实现简单、编译速度快。

此简化规则与调用点适配逻辑协同工作：调用方在决定是否将实参标记为 `e` 时，依据的是调用**后**该值是否仍被使用（调用方视角的活跃性），而非 callee 内部的活跃性。

#### 局部 poly 数组的析构（v2）

当离开局部 poly 数组的作用域时，必须遍历数组中的每一个元素（每个元素均为 `poly` 类型），对每个元素依次调用 `__yesod_poly_free` 释放其堆内存。

- 释放顺序：从后往前（数组末尾到开头），确保不存在 double-free 风险。
- 数组本身的栈空间由正常的栈帧清理完成。
- 若数组中某项在作用域内被移动（如 `arr[i]` 作为返回值移出），该位置在析构前已替换为零多项式（`{nullptr, 0}`），因此 `free(nullptr)` 为安全空操作。

函数参数和返回值中的多项式优先通过寄存器传递两个字段。

内存管理方面，分配块的大小由分配器维护，该分配器提供类似 malloc/free/realloc 的接口，内存块补齐到 2 的幂次。（最小为 8 byte）
- 空闲块通过双向链表连接起来。
- 实现一个基于伙伴算法的分配器，页面大小为 PAGE_SIZE 字节（二的幂，4 KiB 左右），一个 char 数组维护每个页面的阶，未分配则为 0。
  - 对于跨越多个页面的块，只在页面开头存储阶，其他位置存 0。
  - 对于小块，同一个页面只会分配一种大小的块，作为该页面的阶。
  - 无空闲块时分配一个新的页面，划分后全部加入空闲链表（这是最小 8byte 的原因）。小块的内存不会合并。
- 关于 realloc：扩展大小时负责搬运内容，需要传入原大小。缩小空间时，只有实际分配大小至少为目标大小的四倍才会收缩。

## 语法与语义

相对于带有限域类型扩展的增量：

- BType ::= ... | "poly"
- FuncType ::= ... | "poly"
- PrimaryExp ::= ... | PrimaryExp "[" Exp "," Exp "]" | PrimaryExp "[" Exp "]"

### 提取系数

`p[k]`：当 `p: poly, k: int` 时，结果为 `mint`，表示 p 的 k 次项系数。特别地，越界访问（即 $k\not\in [0,n)$）时，返回 `0: mint`。

多项式长度是运行时对象的内部字段，不提供源语言表达式读取。测试和算法若只需要前若干项，必须把项数作为显式参数传递。

### 类型转换

`poly(exp)` 当 `exp: mint` 时，结果为一个只含常数项的多项式（即长度为 1），该常数项恰等于输入的 `exp`。

当 `exp: int` 时，效果相当于两次类型转换的复合：`int -> mint -> poly`

### 切片与移位

`p[n, m]` 当 `p: poly, n, m: int` 时，结果类型为多项式，值为：
- 若 `m <= 0 || n >= m || n >= len(p)`，结果为零多项式，即长度为 0。
- 否则，结果长度为 `min(m, len(p))`，结果的 `n <= i < min(m, len(p))` 次项分别和 `p` 一样，而 `0 <= i < n` 次项为零。

`p << k` 当 `p: poly, k: int` 时，相当于 `p >> (-k)`。
`p >> k` 当 `p: poly, k: int` 时，结果类型为多项式，值为：
- 设 `p` 的长度为 `n`，当 `k >= n` 时结果为零多项式。
- 否则，`k < n` 时结果长度为 `n - k`，其中 `0 <= i < n - k` 次项系数为 `p` 的 `i + k` 次项系数 - 特别地，当 `i + k < 0` 时为零。

### 三则运算

`p + q`, `p - q`：当 `p, q: poly` 时，结果类型为 `poly`，长度为 `p,q` 长度较大值，内容为对位加减、越界补零。

`p * q`：当 `p, q: poly` 时，结果类型为 `poly`，值为：
- 若 `p, q` 之一为零多项式，结果为零多项式。
- 否则，结果为两个多项式的卷积，长度为 `p.n + q.n - 1`

数乘 `p * k` 或 `k * p`：当 `p: poly, k: mint` 时，结果类型为 `poly`，值为：
- 长度保持不变，每一位乘上 `k`

# 代码生成 pass

## 数据流化简

如果常量传播发现一个分支的跳转结果固定，那么把这个基本块和下一个基本块合并。

这个优化加到转化为 SSA 形式之前。

## 所有权分析与“模板”特化

可以和类型分析合并。

每个 `poly` 类型的函数参数都有两种调用约定，每种约定的自由组合构成了函数的一个特化版本。

类似 C++，我们为每个特化版本进行重命名 `__yesod_gen_{name}_{type}`，其中 `{name}` 为原本的名字，`{type}` 为由 r,e 两种字母构成的串，表示每个 `poly` 类型的参数分别采用 calleR/calleE 所有的调用约定，该串的长度和 `poly` 类型参数的数量应保持相同。在所有权分析的过程中，只生成那些可能被使用的特化版本的函数声明与函数定义（不能重复！）。

**特化选择规则（细化版）：** 见上文"传参所有权特化规则（v2）"。

**活跃分析简化：** 对每个函数预生成全 `e` 版本，标记可达 `end` 的基本块，`r` 模式参数在可达 `end` 的块出口处视为活跃。详见上方"活跃分析简化（v2）"。

## 多项式相关 pass

所有这些 pass 都依赖 SSA 形式。

所有这些 pass 中新产生的表达式都要随时常量折叠。

### 多项式非零区间分析（优化 pass）

先直接认为该区间由构造、切片、移位和调用点显式项数约束给出，后续再优化。

正向数据流分析，得到多项式可能非零的区间。

多项式加减法：$range(x\pm y)=range(x) \cup range(y)$

多项式乘法：当 $range(x)=[l_1,r_1],range(y)=[l_2,r_2]$ 时，$range(x y)=[l_1+l_2,r_1+r_2]$；否则只要两个区间有一个为空，结果 $range(x y) =\emptyset$。

多项式数乘：$range(p \cdot k)=range(k \cdot p)=range(p)$

移位：设 $range(p)=[l,r]$ 则 $range(p >> n)=[max(l - n, 0), max(r - n，-1)]$，特判空区间。左移同理。

切片：$range(p[n,m])=range(p) \cap [n,m]$

会随多项式一起作为基本块参数传递给下一个基本块。

### 多项式活跃区间分析（优化 pass）

先直接认为该区间由构造、切片、移位和调用点显式项数约束给出，后续再优化。

逆向数据流分析，得到多项式可能在后续运算中被使用的范围区间 $active$，为 $range$ 的子集。

加减法：$active(x \pm y)\cap range(x) \subset active(x)$

多项式乘法：设 $active(x y)=[l,r],range(y)=[a,b]$ 则 $[l-a,r-a]\cap \range(x)\subset active(x)$

多项式数乘：$active(x k)\cap range(x)\subset active(x)$

移位：设 $active(x >> n)=[l,r]$，则 $[l+n,r+n]\cap range(x)\subset active(x)$

切片：设 $active(x[n,m])\cap range(x)\subset active(x)$

提取系数：提取 $x$ 的 $k$ 次项系数，则 ${k}\cap range(x) \subset active(x)$

如果一个多项式 $p$ 作为基本块参数被传递给后续基本块，则 $active(x)=range(x)$

注意：如果活跃区间包含了尚未计算的值（类型 int），即涉及后续语句中的函数调用、全局变量、数组、后续 SSA 定义的多项式的系数，都需要替换为槑值。
- 槑与任何东西参与运算，结果都是槑。
- $[槑,x]$ 解读为 $(-\infty,x]$，$[x,槑)$ 解读为 $[x,+\infty)$

### 插入 NTT 算子

进行表达式选择，在合适的位置插入 ntt_forward 与 ntt_backward，将表达式划分为系数表示和点值表示两类。
要求：
- 点值表示下只能加减乘（含数乘），不能进行多项式移位、截断等操作。
- 系数表示不能包含多项式乘法

基础算法：直接在多项式乘法前插入 ntt_forward，多项式乘法后插入 ntt_backward

后续可添加的优化：
1. 利用 ntt 的线性性质重写表达式，以减少 ntt 的数量，例如 `ntt_backward(p) + ntt_backward(q) => ntt_backward(p + q)`，`ntt_forward(p * k) => ntt_forward(p) * k`
2. 重写 `ntt_backward(ntt_forward(p)) => p`
3. 合并公共子表达式

### 乘法算子融合与代码生成

对于每个点值表示的连通块，计算所需 ntt 的长度，并生成代码。要求：
- 所有输入先切片到活跃区间，在 ntt_forward 到待定的 ntt 长度 $n$。
- 根据输入的活跃区间，重新计算输出的非零区间。
- 设一个输出新的非零区间为 $[l,r]$，活跃区间为 $[a,b]$。不妨设 $\emptyset\neq[a,b] \subset [l,r]$ 因为 $[a,b]\/[l,r]$ 的差集部分无需计算，可直接赋零。
- 选取最小的 2 的幂 $n$ 使得 $n>max(b-l,r-a)$，这样循环卷积后无关部分不会落在 $[a,b]$ 对应的区间里。

注意：
- 采用 `test/llvm/ntt` 中的参考实现 - 这里用的是转置算法，和普通的 ntt 相比，输出结果的下标是位翻转过的。
- 如果有多个 `ntt_forward(p)`，只需计算其中长度最大的一个，由于位翻转其实长度更短的等于这个的前缀。

### 加法算子融合

现在没有多项式乘法了，首先通过以下恒等式化简多项式为一堆切片+移位的线性组合：
1. $(a\pm b)[n,m] ==> a[n,m]\pm b[n,m]$
2. $(a\pm b)>>n   ==> (a>>n)\pm (b>>n)$
3. $(a \cdot k)[n,m] ==> a[n,m] \cdot k$
4. $(a \cdot k)>>n   ==> (a>>n) \cdot k$
5. $(a \cdot k_1)\cdot k_2 ==> a \cdot (k_1 k_2)$

然后按如下规则重写移位：$a>>k ==> a[k,+\infty]>>k$，这样就可以忽略移位对溢出到负数部分的处理。

之后，按这个规则重写：$(a>>k)[l,r] ==> a[l+k,r+k]>>k, a>>k_1>>k_2 ==> a>>(k_1+k_2), a[l_1,r_1][l_2,r_2] ==> a[max(l_1,l_2),min(r_1,r_2)]$

于是最后式子被改写为标准形式 $\sum_i (a_i[l_i,r_i]>>k_i) cdot c_i$，其中每一个子式的切片、移位、数乘操作至多只有一个。

基础的代码生成算法：为存储结果的多项式分配内存并清零，然后对于每一个子式，扫描切片范围内活跃的每一项，加到结果上。

优化：尝试复用数组：
- 首先筛选子式：
  - 为子式分配的内存大小足够存储结果。
  - 当前语句 $a_i$ 为最后一次使用，之后的语句中不再活跃，且在所有子式中只出现一次。
  - 如果有多个符合条件，优先选择 $c_i=1,k_i=0$ 的；如果还有多个，选择 $r_i-l_i$ 最大的。
- 记录所选子式的编号，用一个 switch 语句来选择对应的分支。每个分支首先将主语句进行原地移位和数乘，多余部分清零，然后把其他子式用相同的方法依次加上来。default 分支内容和基础算法相同。
