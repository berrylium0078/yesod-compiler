## 计划

1. poly_construct 也是值，可以出现在各种命令/基本块参数里。
2. 序列化时，把 poly_construct [a0, a1, ..., an] 简化成 poly { a0, ...., an }

例如：
```koopa
%t_11 = combine ..., (poly {%t_7}, ...)
```

## 循环不变量惰性求值

针对 poly 类型。
1. 为 poly 类型添加特殊值 undef（定义为长度<0）。为 poly 类型添加懒惰计算伪指令 or_else %old, op 其中 op 为任意 poly 类型的指令，语义是若 %old 不为 undef 则返回已经算出的结果 %old，否则返回 op 的计算结果。
2. 找出循环。对于 CFG 回边 A -> B 满足 B 支配 A，将 B->A 所有可能经过的点集视作一个循环，要求该点集的入边都在 B 处（即其他点的入边仍在该点集中）
3. 对每个多项式类型的值，找到外提的目标循环。要求这个值的计算依赖的值的定义最终都可溯源到目标循环外部；找到最外层满足要求的循环。（这里认为基本块参数、load、call 是不可溯源的）
4. 在目标循环前面插入一个新的基本块，为外提目标为该循环的值创建一个 undef 定义，并为该定义迭代插入基本块参数。然后，在目标值定义处使用 or_else 伪指令接受旧定义。

## 多项式拆包

使用伪指令 `get_attr p` 获取多项式的属性，`p_1 = set_attr p, value` 修改多项式的属性。其中 `attr` 是 `base` `addr` `l` `r` 之一。
- `base` 为多项式基址
- `addr` 为多项式实际分配内存块的起始地址
- `n` 为实际分配内存块的长度，为 2 的幂或 0
- `[l, r)` 为多项式的非零区间或活跃区间，取决于优化阶段，用户不可访问。`l >= r` 表示空。
- 保证 `[base + l, base + r)` 是 `[addr, addr + n)` 的子区间

此外，你要为每个 `combine` 或者 `pointwise` 融合算子，生成一个伪指令序列，计算结果的 l, r 属性 - 这相当于提前做了一部分代码生成。

特别地，需要生成 `p` 的 `attr` 属性时，如果发现 `p` 的定义是 `set_attr' p' value`，你需要递归展开定义：若 `attr = attr'` 则返回 `value`，否则返回 `p'` 的 `attr` 属性。

### 多项式非零区间计算

代码生成辅助 pass，计算每个多项式类型的值可能非零的区间，按照上述规则展开定义，且随时常量折叠。

计算规则：
- `a + b` 和 `a - b` 的非零区间为 `a` 和 `b` 的非零区间取并 `[min(l1,l2),max(r1,r2))`，特判空区间。
- 数乘不改变非零区间，除非常量折叠发现乘的是 `0` - 替换成零多项式
- `a * b` 的非零区间为 `[l1+l2,r1+r2-1)`，特判空区间
- `a >> k` 的非零区间为 `[l-k,r-k)`
- `a[x, y]` 的非零区间为 `[max(l,x),min(r,y))`

### 多项式活跃区间计算

计算每个多项式类型的值可能被使用的区间，利用 SSA 形式进行稀疏“分析”。

信息计算规则：（不跨越基本块）
- 若 `a + b` 在 `[l, r)` 上活跃，则 `a` 和 `b` 在 `[l, r)` 上活跃。
- 若 `a >> k`（不考虑负数下标截断）在 `[l, r)` 上活跃，则 `a` 在 `[l + k, r + k)` 上活跃。
- 若 `a[x, y]`在 `[l, r)` 上活跃，则 `a` 在 `[max(x, l), min(y, r))` 上活跃。
- 若 `a * b` 在 `[l, r)` 上活跃，`b` 在 `[x, y)` 上非零，则 `a` 在 `[max(0, l - x), r - x)` 上活跃；对称情况同理。特别地，若 `b` 非零区间为空，则 `a` 的活跃区间为空。
- 对于提取系数操作 `a[k]`，认为 `a` 在 `[k,k+1)` 上活跃。
- 活跃区间最终需要对非零区间取交 - 推迟到最终结算时进行。
- 边界情况：函数调用参数、store、ret 认为活跃区间为槑（非零区间）。

信息合并规则：若 `a` 在 `[l1, r1)` 和 `[l2, r2)` 上活跃，则 `a` 在 `[min(l1, l2), max(r1, r2))` 上活跃。
- 基本块参数 BB(arg = p)：判断是否为回边，回边认为是槑（非零区间），否则若 arg 在 $[l,r)$ 上活跃则也认为 $p$ 在 $[l,r)$ 活跃。由于 SysY 语言没有 goto，删除回边得到的是 DAG，不会死循环。

以上情况假设区间非空，需要对空区间特判。为此请你添加一个 select 伪指令表示无短路求值的条件选择。

然后，以上方法得到的活跃区间可能依赖截至当前融合算子伪指令尚未计算的值。我们尝试把这些值按定义展开，但对于有副作用或计算顺序无法展开的情况 - 例如提取多项式系数、函数返回值、load 内存访问、整数除法 - 认为这些是槑值并传播。特别地，在对非零区间取交时，把左、右端点的槑解读为 `-inf` 和 `+inf`，例如 `[1,槑)` 对 `[2,3)` 取交得到 `[1,3)`。最终结果应该不含槑。

## 循环卷积长度计算

对每个 pointwise 融合算子分别处理。

设这个算子结果的活跃区间为 $[a,b)$。
根据算子的输入活跃区间，重新计算算子输出的非零区间（想象把输入对活跃区间切片），设为 $[l,r)$。

断言 $[a,b) \subseteq [l,r)$，取长度为 $max(r-a,b-l)>=b-a$ 上取整到 $2$ 的幂 - 使用伪指令 `next_pow2` 表示，返回向上取整到 2 的幂后的长度，设为 `len`。特别地活跃区间为空时认为卷积长度是 $0$。

改写算子。
```koopa
%t = pointwise %1 %2 ... %n // 具体运算此处省略
```
改成（示意）：
```koopa
  %len = ... // 一段复杂的表达式(IR 代码块)
  %pv1 = ntt(%1, %len)
  ...
  %pvn = ntt(%n, %len)
  %t = pointwise %len %pv1 %pv2 ... %pvn
  ... // 后续内容
```

## CSE 公共子表达式消除

尝试消除公共子表达式，同时进行一些简单的化简、常量折叠。
- int 或 mint 类型，`0 + x` `x + 0` `x - 0` `1 * x` `x * 1` 化简成 `x`
- `x * 2` 化简成 `x + x` 或移位
- 对于各种 int/mint 运算指令，进行常量折叠，但 div 和 mod 当除数为常数 0 时要保留。
- 如果运算有副作用（例如函数调用、load、store、整数除法或取余）则不消除。目前多项式计算指令都是无副作用的，应该消除。
- 对于跨基本块的情况，只有当两次计算存在支配关系时，消除被支配的一方。
- 如果运算满足交换律（例如加法、乘法、`combine`）则排序 - 包括 `pointwise` 的内部表示。

### 测试

再在 `ASAN_OPTIONS=detect_leaks=1` 下执行。

- 当前基础 SysY/mint LLVM IR 目标为 `x86_64-pc-linux-gnu`。
- 当前 poly LLVM lowering 采用 helper runtime，实现 load/store/copy/setattr
  等硬复制边界；精确活跃区间释放仍为后续优化。
- llvm_basic_test：运行 lv1~lv9
- llvm_extra_test：运行 lvX
- llvm_poly_test：运行 poly 测试集

### 多项式内存分配

LLVM runtime 中 poly 存储为 `base, addr, n, l, r`。`n` 表示已分配内存块的长度，为 2 的幂或 0。

### poly 类型活跃分析

找到每个 poly 类型的值变得不活跃的位置，表示需要回收内存。包括函数参数。

有两种可能：
1. 这个 poly 值的一次 use（不含基本块参数），满足后续没有其他可达 use（含基本块参数）。
   - store, call, pointwise, combine 都是 use。
2. 这个值在基本块 B1 出口处活跃，但在 B1 的一个后继 B2 入口处不活跃 - 其实这说明 B1 以 branch 结尾，且这个值在 B1 另一个后继 B3 入口处活跃。把标记打在 B2 的基本块“调用”处。

测试：Koopa 解释器需要在函数返回时，断言每个局部 poly 对象（包括局部 poly 数组中的每个元素）都被释放恰好一次内存。

### load, store, copy, setattr

load ptr：将 ptr 指向内存空间视作 poly 类型结构体，将该对象硬复制一份出来存放在寄存器（SSA 形式）中。当前 LLVM lowering 调用 poly clone helper 生成硬复制结果。

store ptr, src：释放 ptr 指向内存中的 poly 对象。如果 src 活跃则硬复制；否则移动给 ptr。当前 LLVM lowering 为简单正确性版本，始终把 src clone 后写入目的地址，runtime 在进程退出时统一回收 helper 分配的对象。

dst = copy src：断言 dst, src 均活跃，进行硬复制。（dst 不活跃说明死代码消除没做好；dst 活跃但 src 不活跃说明覆写传播没做好）当前 LLVM lowering 调用 poly clone helper。

result = setattr src, attr, val：断言 src 不活跃，直接翻译成 LLVM IR 的 `insertvalue`。这里使用移动语义，只复制 poly 结构体字段，不复制底层数组。

### 函数调用

函数调用时，将参数移动给 callee，但如果参数仍活跃则硬复制。例如：

```koopa
call @foo(%1, %2, %1, %2) // %1 不活跃，%2 活跃，都是多项式类型
```
需要变成：
```koopa
%1_1 = %1 // 硬复制
%2_1 = %2 // 硬复制
%2_2 = %2 // 硬复制
call @foo(%1_1, %2_1, %1, %2_2)
```

caller 不负责释放 callee 的参数内存。

### 基本块跳转

我们的 SSA 是基本块参数形式的，形式上有些类似函数调用。

`jump` 的处理和函数调用类似。

对于 `branch`，由于多项式对象的复制、释放比较昂贵，进行翻译，插入两个临时基本块。
示意：
```koopa
  branch ... bb1(...) bb2(...)
```
变成：
```koopa
  branch ... bb1_1 bb2_1
bb1_1:
  // 复制和释放 poly 对象
  jump bb1(...)
bb2_1:
  // 复制和释放 poly 对象
  jump bb2(...)
```

如果对基本块 `bb` 的一处“调用”需要在调用前释放内存，将其替换为一个专门释放内存的基本块，这个基本块跳转到 `bb`。

基本块“调用”也是类似的（下一条指令为跳转目标基本块的开头）。

### 数组/全局 poly 对象释放

对于每个全局 poly 数组/全局 poly 变量，在 atexit 时释放。

对于临时栈分配的数组，在 `ret` 前释放。对于 `combine` 或 `pointwise` 融合算子生成的临时数组，在融合算子执行完后释放。

### pointwise 融合算子代码生成

对于指令 `%pvi = ntt(%i, %len)`：
- 若 `%i` 不活跃，且目前分配的内存块大小大于 `len`，进行原地覆写，作为 `%pvi`。
- 否则，为 `%pvi` 分配临时数组并写入。如果 `%i` 不活跃，释放 `%i`。

你可以为两种情况分别创建一个内置辅助函数。注意这里是循环的，即满足 `%pvi[j] = sum_(k % len = j) %i[k]` - 但若 `%1` 的非零区间为 `[l, r)` 则 `r - l >= len * 2` 的情况说明 `%1` 会乘上零多项式，因此 `%pv1` 的值是什么都不影响结果，可以跳过。其他情况可以分类讨论优化。此外还要注意 `addr` 不一定等于 `base`。为了方便调试，你可以让解释器维护动态分配内存的虚拟地址空间，并在分配内存时污染内容防止未初始化导致的错误。

对于 pointwise 指令，生成一个循环，循环体内部开头 `load` 每个 `%pvi` 的 `j` 次项系数，然后进行点值计算，结尾 `store` 结果到 `%pv1.addr + j`。循环结束后，释放 `%pv2, ..., %pvn`，接着对 `%pv1.addr, %len` 调用原地 INTT。

进行后处理：
1. 如果 `%pv1` 的空间利用率低于 $1/4$（活跃区间长度与分配内存块大小的比值），或者因为循环结果不连续，则重新分配内存并复制内容使其连续。
2. 否则，修改 `base` 使得 `[base + l, base + r)` 指向循环卷积的结果。

当前 LLVM lowering 已实现 pointwise 融合循环：为 pointwise 结果分配输出 PV，在 LLVM IR 循环内逐点计算表达式树，随后调用 `__yesod_poly_take_pointwise` 就地 INTT，并在结果区间连续且空间利用率足够时把 PV backing store 转交给 poly，否则重新分配紧凑 poly。第一版仍由 `ntt` helper 生成输入 PV，尚未实现不活跃输入 poly 的 NTT 原地覆写。

### combine 融合算子代码生成

#### 结果内存分配

筛选输入中符合条件的子式，原地覆写作为算子的计算结果：
1. 只出现一次且不活跃（例如 `combine (%p ...) (%p ...)` 中认为 `%p` 出现两次，不符合条件）
2. 数乘系数为 `1`，移位系数为常数 `0`
3. 目前已经分配的内存可以存下结果。设结果的活跃区间为 `[a,b)`，即要求内存区间 `[base+a,base+b)` 是 `[data,data+n)` 的子区间。
4. 如果有多个，选择其中活跃区间最长的

其中，(1) 为编译期计算，(2),(3),(4) 在运行时计算。

如果选择成功，设所选子式在 `[l,r)` 上活跃（已对切片部分取交），整个算子结果的活跃区间为 `[a,b)`，将差集 `[a,b)-[l,r)` 清零。否则分配临时数组并清零。

每一项简化成四个值：
- `l`, `r`：实际被使用的区间
- `k`：乘上的常数
- `src`：基地址

combine 算子主体部分伪代码（仅用于说明四个值的含义，`dst` 表示存储结果的数组）：
```c
for (int i = 0; i < n; i++)
  for (int j = l[i]; j < r[i]; j++)
    dst[j] += src[i][j - l[i]] * k[i];
```

#### 优化 combine 算子

1. 函数内联
1. 去除所有 `k[i]=0` 的项，把所有项按 `k[i]` 分成 `1`，`-1` 和其他，总共三类。
2. 将所有区间端点排序，扫描线算法维护每一段有哪些项参与运算。
3. 对于每一段 `[L,R)`，展开成一个循环。示意伪代码：
   ```c
   mint *p = dst + L, *endp = dst + R;
   mint *p1 = src1 + L - offset1; // k1 = 1
   mint *p2 = src2 + L - offset2; // k2 = ?
   mint *p3 = src3 + L - offset3; // k3 = -1
   while (p != endp) {
     // 加载
     mint x = *p, x1 = *p1, x2 = *p2, x3 = *p3;
     // 数乘
     x2 = x2 * k2;
     // 累加（分治），符号和区间的左端点保持一致，例如 -x5-x6 => t6=x5+x6
     mint t1 = x + x1;
     mint t2 = x2 - x3;
     mint t3 = t1 + t2;
     // 存储
     *p = t3;
     // 循环变量自增
     p++; p1++; p2++; p3++;
   }
   ```
4. 特别地，`mint` 数乘先不运行蒙哥马利 reduce。固定表达式形式，维护每个和式的数值范围，在即将 `int64` 溢出时通过便宜的比较、加减、cmov 运算将范围折半。最终把范围缩进 `M << 32` 后再 reduce。
5. 对于很长的段，可以尝试拆成长度能放进缓存的小段。

#### 内存清理

释放所有不活跃输入 `poly` 对象。注意不要重复释放，不要释放重新利用为结果的那个输入。
